// Notas del flow actual
· ?play=creator es la entrada canónica (bookmark-friendly).
· NUEVO r1: modal con 4 built-in templates + custom user-saved (localStorage). Antes el Creator entraba directo al editor vacío sin alternativa.
· Plantillas tienen config (grid+5 colores), layout (mobiliario) y opcional staffPos. applyXpaceTemplate() sobreescribe FACTORY_LAYOUTS.creator + slot.current de localStorage antes de que initGame() → loadLayout() corra.
· El single xpace_creator_template legacy sigue como fallback de Shoptalk u otros Xpacios en blanco.
· BUG detectado durante la captura: al hacer click en una tarjeta built-in (ej. 🚬 Tabaquería), el modal se cierra y el editor abre, pero shopLayout queda en 0 items en lugar de cargar la plantilla. applyXpaceTemplate escribe en FACTORY_LAYOUTS.creator y persiste en localStorage, pero algo del flow enterXpaceCreator → proceed → initGame → loadLayout termina ignorándolo. Pendiente fix en r2.

// Ideas para r2 y siguientes
· Fix del bug applyXpaceTemplate arriba.
· Preview miniatura en cada tarjeta del modal (canvas SVG/mini-render del layout en vez de solo icon).
· Compartir plantilla: subir al worker (R2 + endpoint /creator/template) → URL pública.
· Edit plantilla: load + edit + save reemplazando la original (ahora siempre crea nueva).
· AI assist: brief textual → LLM genera {config, layout} automáticamente.
· Multi-planta / multi-piso: ahora 1 nivel.
